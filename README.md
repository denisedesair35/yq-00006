# 本地资料整理小工具

一个轻量级的本地文件整理工具，帮助你快速扫描、统计和标记文件状态。

## 功能特性

- 📂 **文件夹扫描**：输入本地文件夹路径，快速扫描所有文件
- 📊 **多维度统计**：按文件类型、修改时间、文件大小进行分类统计
- 🏷️ **状态标记**：为每个文件标记「待处理、已整理、忽略」三种状态
- 💾 **状态持久化**：状态自动保存到本地 JSON 文件，下次打开还在
- 🔍 **筛选搜索**：支持按类型、状态、关键词筛选和搜索文件
- 📋 **批量操作**：可多选文件批量设置状态

## 技术栈

- **后端**：Node.js + Express
- **前端**：原生 HTML / CSS / JavaScript
- **数据存储**：本地 JSON 文件

## 启动说明

### 环境要求

- Node.js >= 14.0.0

### 安装依赖

```bash
npm install
```

### 启动服务

```bash
npm start
```

启动成功后，终端会显示：

```
========================================
  本地资料整理小工具 - 服务已启动
========================================
  访问地址: http://localhost:3000
  服务端口: 3000
========================================
```

### 访问页面

打开浏览器访问：**http://localhost:3000**

## 验证说明

### 1. 基础功能验证

1. 启动服务后，在浏览器打开 http://localhost:3000
2. 在输入框中填写一个本地文件夹路径（例如：`D:\我的资料` 或 `C:\Users\你的用户名\Downloads`）
3. 点击「扫描文件夹」按钮
4. 预期结果：
   - 显示统计概览（文件总数、总大小、各状态数量）
   - 显示按类型、时间、大小的分类统计
   - 下方显示文件列表

### 2. 状态标记验证

1. 在文件列表中，点击任意文件右侧的「已整理」或「忽略」按钮
2. 观察：
   - 文件状态徽章会立即更新
   - 顶部统计数字会同步变化
3. 刷新浏览器页面，重新扫描同一文件夹
4. 预期结果：之前设置的状态仍然保留（证明数据持久化有效）

### 3. 筛选搜索验证

1. 在搜索框输入关键词，观察列表实时过滤
2. 在「文件类型」下拉框选择某一类型，验证只显示该类型文件
3. 在「状态筛选」下拉框选择「已整理」，验证只显示已整理的文件
4. 在「排序方式」下拉框切换排序方式，验证列表顺序变化

### 4. 批量操作验证

1. 勾选多个文件前的复选框（或点击「全选」）
2. 点击列表上方的批量操作按钮（如「批量设为已整理」）
3. 预期结果：选中文件的状态批量更新

### 5. API 接口验证（可选）

可以直接通过浏览器或 curl 测试接口：

```bash
# 健康检查
curl http://localhost:3000/api/health

# 扫描文件夹（Windows 路径示例）
curl -X POST http://localhost:3000/api/files/scan \
  -H "Content-Type: application/json" \
  -d "{\"folderPath\":\"D:\\\\我的资料\"}"
```

## 数据存储

- 文件状态存储在 `data/file-status.json`
- 以文件夹路径为 key，文件完整路径为 value 存储状态
- 删除该文件可重置所有状态
